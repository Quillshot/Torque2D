T2D A-Star Path Finding

 

Overview

 

 

T2D 1.7.3 introduced an integrated A-Star path finding implementation designed to work directly with your level. As an introductory feature, the AStar implementation provides the key capabilities needed for many styles of games that benefit from dynamic path finding. The initial implementation was based off the excellent work performed by Phil Shenk, as first released in his T2D resource A-Star Pathfinding Extension for T2D. Some of the features have been streamlined and reconfigured for binary only users, and a few capabilities added or adjusted in this release, providing additional flexibility. Many Thanks to Phil for his outstanding work!

 

A-Star Concepts

 

A-Star Path Finding is a common computer AI requirement that allows developers to create a set of data within their game world to enable agents to navigate within the world in a (relatively) smart manner. It is based on a powerful and flexible algorithm designed around the concept of a grid that contains values indicating how much "effort" it requires to navigate across a grid square, allowing objects to make decisions about what path to take to reach a destination.

 

A-Star calculations are relatively expensive compared to other aspects of a game engine, and it's important to realize from the beginning that they don't have unlimited power-the size of a grid, and therefore the total number of navigation points within the grid, directly impacts the time it takes to calculate an A-Star path. Careful selection of a navigation grid size is very important for every game and is dependent on a number of factors, so it is important to realize that you can only go so far with how you use A-Star navigation before you run into performance issues.

 

There are many different strategies and techniques for optimizing A-Star Path Finding, and while we've selected a basic feature set that balances functionality and performance, some games may require more capability, or more performance. A Torque 2D source code license allows you to modify the underlying algorithms, and as the T2D A-Star implementation matures we will be leveraging requests and suggestions from the Torque Community to make this feature as powerful and robust as feasible for our users.

 

Torque 2D A-Star Implementation

 

As Phil describes in his resource, T2D Path Finding is based around two key concepts:

 

Integration with t2dTileMap objects: While the AStar grid and paths are independent classes within T2D, they utilize Tile Maps and Tile Layers to provide the weight data, allowing users to leverage the existing Tile Map editor interface for custom grid development. Users create a Tile Map and Tile Layer to represent their path finding grid, and create brushes with custom data values to define the weights of the grid itself. When an A-Star path is desired, the built in classes access the data within the Tile Layers to generate their paths.

To create a grid with predefined path information, you should use the T2D tile map editor, and define any brushes meant to represent tiles with travel costs via the customData field.

  • CustomData value of 0: No cost to travel
  • CustomData value of 10: tile is not passable
  • CustomData value between 0-10: increased cost of travel, which will control if this tile is used based on relative path weights of the nearby tiles

 

Powerful TorqueScript Interface: All of the major A-Star functionality is managed via TorqueScript, with the ability to create and utilize A-Star paths with no C++ code. This implementation also allows aStarActors to use T2D's moveTo() capability to control how objects within the scene navigate across the A-Star grid based on information provided by the createPath() script call.

 

Torque 2D A-Star Objects (C++)

 

There are two classes that implement the A-Star functionality within T2D, both of which are fully accessible via TorqueScript:

 

pathGrid2d: This class represents the data grid used for aStarActors to interact with the navigation grid. It provides the following key functionalities:

  • Tile Layer Integration-access to the T2D Tile Layer that defines the weights for the navigation grid.
  • Path Creation-allows aStarActors to provide a start point and an end point, and calculates a list of nodes to navigate to the destination
  • Path Management - stores all paths that have been created for this grid, and allows for both creation and deletion of A-Star paths.

 

pathAStar2d: This class represents an individual path that has been created based upon the grid data. It allows for:

  • Creating and Deleting paths-using the data within the pathGrid, aStarActors can individually create their own paths, and use the list of nodes within the path to navigate through the game world.
  • Accessing path information-aStarActors can either use the stock navigation harness to traverse the path, or can query the path directly for more advanced navigation, such as spline based movement.

 

Torque 2D A-Star Script NameSpaces (TorqueScript)

 

Torque 2D also provides a TorqueScript interface to the underlying implementation that allows for easy use of the navigation grid without delving deep into the A-Star implementation. Individual T2D objects can be assigned a class or superclass of aStarActor, which manages higher level needs for developers, including:

 

  • Management of actor states-notification and handling of basic AI dependent on the validity of assigned paths, world change notifications (dynamic weight assignment to the grids), and path creation/deletion management. Also provides for Active, Frustrated, and Dead states for aStarActors.
  • Integration with T2D moveTo()-utilizing the path information provided by the pathAStar2d class, aStarActors can control how they move along a list of nodes, including callback notification for each node achieved.
  • World change notification-handles dynamic (real time) changes to the aStarGrid, and re-calculates paths based on world state changes.

 

T2D also provides utility functions for working with the pathGrid2d C++ class via the pathGrid2d namespace, including:

 

  • Creation of and coupling with T2D Level Builder Tile Layers identified as a PathLayer.
  • Grid weight access methods for modifying and retrieving navigation grid weights based on either world coordinates or grid index pairs.
  • Interface with the aStarActor namespace for notification of world change events.
