Using Dynamic Fields

Dynamic fields let you store data associated with a T2D object. In order to work with an object's dynamic fields, open up the object's Dynamic Fields rollout in the Edit tab (you may need to scroll down to find it). For an object with no dynamic fields defined, its Dynamic Fields rollout will look like Figure 1.


Figure 1


The Field Name will hold the name of the dynamic field we want to create, and the Field Value is the actual value we want to store in the field. For our example, we will add an hSpeed and vSpeed value, representing horizontal and vertical speeds for an object. First, enter “hSpeed” into the Field Name section, then enter “30” into the Field Value section (as shown in Figure 2). For the tutorials, add whatever fields and values it suggests.



Figure 2

Next, click the green plus sign button to confirm that you want to add the field. You should now see “hSpeed” as non-editable text, and the value as editable with a little red minus sign instead of the plus sign (as shown in Figure 3). If you want to delete a dynamic field, click the minus sign. For an example, I'll add vSpeed. Type “vSpeed” into the Field Name and “15” into the Field Value. Then click the little green plus sign, and the vSpeed should appear just like the hSpeed (as shown in Figure 4).
 
Figure 3
Figure 4

 

 

This system allows you to add any custom data you want to your T2D game objects. In script, they can be accessed with the dot operator, like so:
%this.vSpeed = 20;
