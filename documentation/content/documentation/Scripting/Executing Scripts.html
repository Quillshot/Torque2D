Exec and You

To execute a script,  simply add the following code to your game.cs script in <your T2D Game Projects Directory>/<your T2D Game>/game/gameScripts/game.cs :

 
exec("./aScript.cs");

Obviously, change "aScript" to the name of the script you want to execute.

You probably want to put this line at the top of the startGame function - this way, your script changes will be recompiled each time you run the game.

Brief Explanation of Exec:

Because game logic should be spread across multiple script files (putting it all in one file would be a bad idea), TorqueScript needs a way to tell which scripts need to be prepared for your game. When the file is exec'd, it will be compiled into a .dso if needed and the file's code will be ready for use with the rest of your game.

As noted above, for changes to a script to show up in your game, they need to be exec'd. For this reason, exec statements are often put in the startGame function in game.cs. startGame is called by the Level Builder every time you hit play, so the scripts will be recompiled if needed. game.cs itself, however, is only recompiled when you open that project, so many people simply close T2D and reopen it (as many of the tutorials do).

Remember exec the next time your script changes aren't taking effect.
